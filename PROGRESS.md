# latexd Progress

## Current Status

- Scope: Rust-based partial LaTeX-compatible incremental compiler, following the roadmap in [`README.md`](/home/seorii/dev/hancomac/latexd/README.md).
- Detailed remaining-work backlog: [`docs/work-backlog.md`](/home/seorii/dev/hancomac/latexd/docs/work-backlog.md).
- Detailed `M12` checklist and definition of done: [`docs/m12-checklist.md`](/home/seorii/dev/hancomac/latexd/docs/m12-checklist.md).
- Milestone split: `M11` is tracked as `M11.1`~`M11.4`, and `M12` is tracked as `M12.1`~`M12.6`.
- Current phase: post-`M12` follow-on planning on top of completed `M11` and completed `M12`. The project now has an end-to-end daemon/web preview slice, a project model, dependency-aware rebuild skipping, a tokenizer/interner, a small TeX VM, format snapshots, local `.cls`/`.sty` loading, an internal text layout/PDF pipeline, checkpoint metadata, page-level artifact reuse, a viewport-tile preview path that can run against the mock rasterizer, `gs`, or `libgs`, a line-aware syncmap/click-selection preview path with page-local source/output windows and explicit open-source bridge surfaces, and a semantic aux subsystem that persists concrete artifacts, drives rerun-to-fixpoint, and participates in checkpoint/page reuse.

## Known Gaps

- No real TeX box/glue/page builder yet. Layout is text-based, fixed-width, and only suitable as an internal scaffold.
- No real LaTeX kernel bootstrap yet. The mini kernel is a handcrafted subset.
- Semantic aux is still a scaffold, not a full LaTeX-compatible aux engine.
  - the current path resolves a narrow set of source commands by materializing rewritten inputs, rerunning to a semantic fixpoint, optionally seeding from a reusable preamble checkpoint, and backdating equal aux payloads across revisions, but it does not model real `.aux` bytecode, BibTeX/Biber execution, package-specific citation styles, or arbitrary TeX macro-generated labels/citations.
- Shipout replay is still conservative. Toplevel edits now use saved shipout source offsets directly and can also reuse same-file `\input` exit checkpoints, while non-toplevel `\input` edits replay from stored input-boundary checkpoints with nested enter/exit continuation frames and source-ranged module traces, but dirty-region selection is still far from a trace-grade VM/input dependency model and repeated include instances are only conservatively grouped by continuation context.
- No true vector/page-layer renderer yet. The viewer now renders a page stack of per-page SVG/PDF artifacts and can overlay viewport tiles on visible pages, but it still relies on simple asset swapping rather than a richer page scene graph.
- No persistent-process Ghostscript adapter yet. `tex-render-gs` now has working `gs-cli` and minimal reusable `gs-api` backends, and latexd caches/deduplicates the resulting rasters in memory and on disk, but both backends still pay `init_with_args`/device setup work for a truly cold miss when neither cache is warm.
- Source attribution is still scaffold-grade rather than true layout-engine sync.
  - It now respects nested input boundaries, output order, page-local output ranges, line-aware box geometry, and an in-browser source pane with revision-stable source fetch, but it is still derived from text spans rather than true box-level sync data from a real TeX layout engine.
